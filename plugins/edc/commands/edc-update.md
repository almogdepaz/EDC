# Update Context

**Arguments:** $ARGUMENTS

Parse arguments:
1. **Base** (optional): `--base <ref>` for comparison reference. Default: auto-detect merge base with main/master.

## Prerequisites

`.context/full-context.md` and `.context/.meta.json` must exist. If not, tell the user to run `/edc:edc-build` first.

## Process

### Step 1 — Detect what changed

```bash
# Auto-detect base if not provided
BASE=$(git merge-base HEAD main 2>/dev/null || git merge-base HEAD master 2>/dev/null)

# Get changed files
git diff --name-only $BASE..HEAD
```

Read `.context/.meta.json` to get the last analyzed commit and module→file mappings.

### Step 2 — Identify affected modules

Map each changed file to its module using the `modules` mapping in `.meta.json`. Collect the set of affected modules. Also include any modules that `.context/{module}.md` documents as coupled to an affected module (read the cross-module coupling sections).

### Step 3 — Re-analyze affected modules

For each affected module, spawn a **clean subagent** (using the Agent tool) to invoke the `edc:edc-context` skill (NOT `audit-context-building` — that is a different plugin) on ONLY the files in that module. The subagent MUST be fresh with NO access to the current conversation context. Read the changed files and their immediate dependencies. Produce updated analysis for that module.

**Clean Slate Rule:** Analysis subagents must NOT inherit the parent conversation to prevent bias. See edc-build.md for rationale.

### Step 4 — Update .context/{module}.md files

Rewrite each affected module's `.context/{module}.md` with the new analysis. Preserve the file format (`<!-- generated by /edc:edc-build -->` header).

### Step 5 — Update .context/issues.md

Re-read all `.context/{module}.md` files. Rebuild issues.md from scratch:
- Keep issues that still exist in the code
- Remove issues that were fixed by the changes
- Add new issues found in the updated modules
- Preserve issue numbering where possible for traceability

### Step 6 — Update context.md if needed

Re-read all `.context/{module}.md` files. Check if any of these changed:
- module list (modules added or removed)
- global invariants
- trust boundaries
- cross-module coupling
- key flows

If any changed, rewrite `context.md`. If not, leave it alone.

### Step 7 — Update full-context.md

Append a section to `.context/full-context.md` documenting this incremental update:
```
---
## Incremental Update — <date>
Base: <commit>
Head: <commit>
Changed files: <list>
Modules re-analyzed: <list>
Issues added: <count>
Issues removed: <count>
```

### Step 8 — Update .meta.json

Write updated `.context/.meta.json`:
```json
{
  "lastCommit": "<HEAD commit SHA>",
  "lastFullBuild": "<original full build commit>",
  "lastUpdate": "<ISO timestamp>",
  "modules": {
    "module-name": {
      "files": ["src/path/file.ts", ...],
      "contextFile": ".context/module-name.md"
    }
  }
}
```
